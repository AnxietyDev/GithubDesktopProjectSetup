Team Onboarding Guide

TheEnchanter

Setting up GitHub Desktop for the project

⚠ Before You Begin
You must have been added as a collaborator on the GitHub repo before any of this will work. Ask your lead to add your GitHub username to the project first.
Installation & Setup
I

Download GitHub Desktop

Download and install GitHub Desktop from desktop.github.com. Once installed, sign in with your GitHub account from File → Options → Accounts.

Note
Use the same GitHub account you were added to the repo with. If you have multiple accounts, double-check which one is active.
II

Install Git LFS

Git LFS handles the large binary files in the project (Blueprints, maps, textures). Without it your clone will be broken. Download from git-lfs.com, install it, then open Git Bash and run:

git lfs install

You only need to do this once per machine.

III

Clone the Repository

In GitHub Desktop go to File → Clone Repository → URL and paste the repo URL. Set your local path to somewhere on your machine — ideally your D: drive if you have one, as UE projects get large.

# Recommended local path D:\GitHub\TheEnchanter

Click Clone and wait for it to finish — LFS assets may take a few minutes to download.

Daily Workflow
IV

Always Fetch Before You Start

Every time you sit down to work, hit Fetch origin at the top of GitHub Desktop before opening Unreal. This pulls any changes your teammates pushed since you last worked.

Why this matters
Opening UE before fetching means you might start editing an asset someone else already changed. Merging binary files is impossible — one person's work gets lost.
V

Lock Files Before Editing

Before editing any .uasset or .umap file, lock it so nobody else edits it at the same time. Open Git Bash in the project folder and run:

git lfs lock Content/Path/To/YourFile.uasset # example git lfs lock Content/Maps/DungeonLevel01.umap

When you're done and have pushed your changes, unlock it:

git lfs unlock Content/Maps/DungeonLevel01.umap
VI

Committing Your Work

When you're ready to save your work to the repo, go to GitHub Desktop. You'll see your changed files listed on the left. Write a commit message in the box at the bottom — this is mandatory and must follow the format below:

# Format [Category] Short description of what changed # Examples [BP] Added stamina drain to sprint mechanic [Map] Blocked out west wing of dungeon level 1 [Art] Added texture set for stone floor tiles [Fix] Fixed player respawn spawning underground
TagUse for
[Art]Meshes, textures, materials, VFX
[Audio]Sound effects, music, dialogue
[BP]Blueprint changes
[Code]C++ source changes
[Map]Level / map edits
[UI]Widget or HUD changes
[Anim]Animation assets or AnimBPs
[Fix]Bug fixes
[Config].ini or project settings
❌ Not acceptable
Commit messages like update, fix, changes, wip, or assets tell your teammates nothing. These will be asked to be redone.
VII

Never Commit These Folders

These folders are already in .gitignore and should never appear in your changes list. If they do, something is wrong — ask your lead before committing:

# These are auto-generated and must stay out of the repo Binaries/ Saved/ Intermediate/ DerivedDataCache/ Build/
VIII

Push & Unlock

After committing, click Push origin in GitHub Desktop to send your changes to the remote repo. Then unlock any LFS files you locked earlier.

Reminder
Don't hold locks overnight or across sessions. If you lock a file and forget to unlock it, nobody else on the team can edit it until you do.
